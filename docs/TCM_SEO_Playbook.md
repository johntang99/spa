## 2. The Pipeline Command

### Prerequisites

1. `content/<site-id>/intake.json` exists with business info + services
2. `.env.local` has `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`
3. `site_seo_pages` table exists in Supabase
4. `[slug]/page.tsx` dynamic route is deployed

### Commands

```bash
# Generate all SEO pages (EN + ZH) + service pages
node scripts/seed-seo-pages.mjs <site-id>

# Verify everything
node scripts/verify-site.mjs <site-id>

# Full pipeline with QA + report
node scripts/seo-pipeline.mjs <site-id>

# All sites at once
node scripts/seed-seo-all-sites.mjs
```

## 3. Step-by-Step Execution (6 Phases)

### Phase 1 — Generate SEO Pages (Automated, ~5 min)

```bash
node scripts/seed-seo-pages.mjs <site-id>
```

**What it does:**
- Reads `intake.json` for business info, services, city
- Reads `site.json` + `contact.json` for real address, phone, hours, map
- Calls Claude API → generates EN content for 5 page types (landing + 3 conditions + cost)
- Calls Claude API → generates ZH content
- Calls Claude API → generates 1 service page per modality (4 calls, EN + ZH each)
- Upserts all pages into `content_entries`
- Registers all slugs in `site_seo_pages`

**Output:** 9 slugs × 2 locales = 18 pages

### Phase 2 — Add SEO Objects to Existing Pages (~5 min)

Update `content_entries` DB for `pages/home.json`, `pages/about.json`, `pages/services.json`, `pages/contact.json` to add `seo` field:

```javascript
// For each page:
row.data.seo = {
  title: "...",           // ≤60 chars
  description: "...",     // ≤155 chars
  canonicalUrl: "/en/...",
  schema: ["..."],
  noindex: false,
  priority: 0.8
};
```

### Phase 3 — Internal Link Rewiring (~15 min)

**This is the most important manual step.** All existing pages must link to SEO pages.

#### 3.1 Homepage — Services Section

Update `pages/home.json` → `services.services[].link` and `services.featured.link`:

| Service ID | Old Link | New Link |
|-----------|----------|----------|
| acupuncture (featured) | `/en/services#acupuncture` | `/en/acupuncture-{city}-{state}` |
| herbal-medicine / herbs | `/en/services#herbs` | `/en/chinese-herbal-medicine-{city}-{state}` |
| cupping | `/en/services#cupping` | `/en/cupping-therapy-{city}-{state}` |
| moxibustion | `/en/services#moxibustion` | `/en/moxibustion-{city}-{state}` |
| tuina | `/en/services#tuina` | `/en/tui-na-massage-{city}-{state}` |

#### 3.2 Homepage — Conditions Section

Update `pages/home.json` → `conditions.conditions[].link`:

| Condition | Old Link | New Link |
|-----------|----------|----------|
| Back Pain / Pain | `/en/conditions#pain` | `/en/acupuncture-for-back-pain-{city}-{state}` |
| Anxiety | `/en/conditions#anxiety` | `/en/acupuncture-for-anxiety-{city}-{state}` |
| Insomnia / Sleep | `/en/conditions#sleep` | `/en/acupuncture-for-insomnia-{city}-{state}` |
| Others | anchor links | stay as-is (no SEO page yet) |

#### 3.3 Services Page

Update `pages/services.json` → `servicesList.items[].link`:

| Service ID | Link |
|-----------|------|
| acupuncture | `/en/acupuncture-{city}-{state}` |
| chinese-herbal-medicine | `/en/chinese-herbal-medicine-{city}-{state}` |
| cupping-therapy | `/en/cupping-therapy-{city}-{state}` |
| moxibustion | `/en/moxibustion-{city}-{state}` |
| tuina-massage | `/en/tui-na-massage-{city}-{state}` |

> The `ServicesSection` component renders "Learn More →" links when `service.link` exists.

#### 3.4 Conditions Page

Update `pages/conditions.json` → `conditions[].link`:

| Condition keyword in title | Link |
|---------------------------|------|
| back pain, pain, neck, arthritis | `/en/acupuncture-for-back-pain-{city}-{state}` |
| anxiety, panic, stress | `/en/acupuncture-for-anxiety-{city}-{state}` |
| insomnia, sleep | `/en/acupuncture-for-insomnia-{city}-{state}` |

#### 3.5 Homepage seoLink

```javascript
home.data.services.seoLink = {
  url: '/en/acupuncture-{city}-{state}',
  text: 'Acupuncture in {City}, {State}'
};
```

#### 3.6 Header Navigation

Add nav item after "Home":
```javascript
{ url: '/en/acupuncture-{city}-{state}', text: 'Acupuncture' }
```

#### 3.7 Footer Columns

```javascript
footer.data.resources = [
  { url: '/en/acupuncture-cost-{city}-{state}', text: 'Acupuncture Cost' },
  { url: '/en/acupuncture-{city}-{state}', text: 'Acupuncture in {City}' }
];
footer.data.serviceAreas = [
  { url: '/en/acupuncture-{city}-{state}', text: '{City}, {State}' },
  { url: '/en/acupuncture-for-back-pain-{city}-{state}', text: 'Back Pain Treatment' },
  { url: '/en/acupuncture-for-insomnia-{city}-{state}', text: 'Insomnia Treatment' },
  { url: '/en/acupuncture-for-anxiety-{city}-{state}', text: 'Anxiety Treatment' }
];
```

## 4. Internal Link Rewiring — Full Checklist

Use this checklist for every new TCM site. Every item must be completed.

```
Homepage:
  □ services.featured.link → core landing page
  □ services.services[].link → each service SEO page
  □ conditions.conditions[].link → each condition SEO page (where available)
  □ services.seoLink → core landing page

Services page:
  □ servicesList.items[].link → each service SEO page (5 services)

Conditions page:
  □ conditions[].link → each condition SEO page (3 conditions)

Header:
  □ Nav item "Acupuncture" → core landing page (after "Home")

Footer:
  □ Resources column → cost page + core landing page
  □ Service Areas column → core + 3 condition pages

SEO pages (generated):
  □ Core landing → links to all condition + service pages (built into layout)
  □ Condition pages → links to sibling conditions + core landing (built into layout)
  □ Service pages → back-link to core landing (built into layout)
  □ Cost page → links to core landing (built into layout)
```

## 6. Content Requirements Per Page Type

### Core Landing Page (`seo-local-landing`)

| Section | Content |
|---------|---------|
| Hero | H1 + city, subheading, intro, CTA, trust bar |
| Conditions grid | 6-8 conditions linking to condition pages |
| Services | 5 service cards linking to service pages |
| Why Choose Us | 3-4 trust points + testimonial |
| FAQ | 6 questions with FAQPage schema |
| Location | Map, NAP block, hours, CTA |

### Condition Page (`seo-condition`)

| Section | Content |
|---------|---------|
| Hero | H1, opening paragraph (100-130 words), CTA |
| How It Works | Mechanism + types treated |
| What to Expect | Step-by-step treatment process |
| Patient Story | 1 testimonial mentioning condition + city |
| FAQ | 4 questions with FAQPage schema |
| Related Conditions | Links to sibling condition pages |

### Service Page (`seo-service`)

| Section | Content |
|---------|---------|
| Hero | H1 + service name, description, CTA |
| What Is It | 2-3 paragraphs explaining the modality |
| Conditions Treated | 6 conditions this service addresses |
| How It Works | Treatment process, what to expect |
| FAQ | 4 questions specific to this service |
| CTA | Back-link to core landing + booking CTA |

### Resource Page — Cost (`seo-resource`)

| Section | Content |
|---------|---------|
| Hero | H1 (question), price range in first paragraph |
| Price Breakdown | Table of session types and prices |
| What Affects Cost | Factors that change pricing |
| Insurance | Coverage info, HSA/FSA |
| Worth It | ROI framing + testimonial |
| FAQ | 5 questions with FAQPage schema |

## 7. AEO (AI Search) for TCM

### Monthly AI Citation Monitoring Queries

Search these in ChatGPT + Perplexity (replace {city} and {practitioner}):

```
1. "Best acupuncturist in {city}"
2. "Acupuncture for back pain near {city}"
3. "How much does acupuncture cost in {state}?"
4. "Does acupuncture work for insomnia?"
5. "{practitioner name} acupuncture"
```

### TCM-Specific AEO Rules

- Always include "What is [service]?" section on service pages
- Use medical-accuracy language: "may help", "has been shown to support"
- Include specific numbers: session count ranges, pricing ranges
- FAQ schema on every money page
- Keep content fresh — update prices and stats quarterly

## 8. GSC + GBP + Reviews Playbook

### Google Search Console

1. Verify property for the domain
2. Submit sitemap: `https://{domain}/sitemap.xml`
3. Request indexing for all 9 SEO URLs
4. Check monthly: impressions, clicks, position for target keywords

### Google Business Profile

| Action | Details |
|--------|---------|
| NAP consistency | Business name, address, phone must match website footer exactly |
| Description | 700+ characters, include city + primary keywords |
| Services | Add: Acupuncture, Herbal Medicine, Cupping, Pain Management, Sleep, Anxiety, Fertility |
| Photos | 12+ photos: clinic exterior, treatment rooms, practitioner, herbal pharmacy |
| Q&As | Seed 5-7 from FAQ content on SEO pages |
| Posts | 1/week — treatment tips, seasonal health, condition spotlights |

### Review Collection

| Step | Action |
|------|--------|
| 1 | Create Google review link shortcut |
| 2 | Draft email template: "Thank you for visiting — would you share your experience?" |
| 3 | Send review request 2 days after appointment |
| 4 | Target: 25+ reviews within 60 days |
| 5 | Respond to ALL reviews within 48 hours |

